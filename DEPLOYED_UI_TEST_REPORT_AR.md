# تقرير اختبار النسخة المنشورة على botzo.net

تاريخ الاختبار: 2026-05-25

## ملخص النتيجة

النسخة المنشورة تعمل الآن من ناحية السيرفر والدومين وPHP وقاعدة البيانات، لكن فلو التسجيل ليس جاهزًا بالكامل لأن صفحة التسجيل ترجع 404. كما أن النسخة المنشورة على `/home/botzkprp/botzo.net` ليست بالضرورة نفس آخر كود موجود في GitHub داخل `/home/botzkprp/repositories/botzoo`.

## نتائج الاختبار

| الفحص | النتيجة |
|---|---|
| الصفحة الرئيسية `https://botzo.net/` | تعمل `200 OK` |
| صفحة الدخول `https://botzo.net/login` | تعمل `200 OK` |
| صفحة التسجيل `https://botzo.net/register` | لا تعمل `404 Not Found` |
| اللغة الحالية `/current-locale` | تعمل وترجع `ar` |
| اتجاه الواجهة | `rtl` |
| جاهزية النظام `/health/ready` | جاهز `ready` |
| قاعدة البيانات | تعمل |
| PHP | يعمل على `PHP/8.2.31` |

## اختبار حساب أحمد

تمت محاولة تسجيل الدخول بالحساب:

```text
ahmed@gmail.com
```

والنتيجة:

```text
بيانات الدخول المقدمة غير صحيحة.
```

هذا يعني أن الحساب غير موجود في قاعدة الإنتاج الحالية أو أن كلمة المرور مختلفة.

## مشاكل مؤكدة

### 1. صفحة التسجيل لا تعمل

المسار:

```text
https://botzo.net/register
```

يرجع:

```text
404 Not Found
```

مع أن الواجهة تعرض رابط/زر "إنشاء حساب"، لذلك فلو التسجيل من البداية غير جاهز للمستخدم النهائي.

### 2. النسخة المنشورة مختلفة عن الريبو

الكود الذي يتم تشغيله فعليًا موجود هنا:

```text
/home/botzkprp/botzo.net
```

بينما GitHub تم عمل clone له هنا:

```text
/home/botzkprp/repositories/botzoo
```

لذلك أي تعديل يتم رفعه إلى GitHub لن يظهر على الموقع إلا بعد مزامنة الكود أو اعتماد مسار نشر واضح.

### 3. تعديل .htaccess كان ضروريًا على السيرفر

كان يوجد سطر PHP handler تسبب في منع تشغيل ملفات PHP وإرجاع 403:

```apache
AddHandler application/x-httpd-ea-php82 .php .php8 .phtml
```

تمت إزالته على السيرفر، وتم تعديل نسخة الريبو أيضًا حتى لا ترجع المشكلة عند النشر القادم.

## حالة البريد

إذا كان ملف `.env` ما زال يستخدم:

```env
MAIL_MAILER=log
```

فلن تصل رسائل التفعيل إلى البريد الحقيقي. يجب ضبط SMTP أو Resend قبل اختبار التفعيل الحقيقي.

## المطلوب قبل اختبار الفلو كاملًا

1. إصلاح مسار التسجيل أو تفعيل route التسجيل في النسخة المنشورة.
2. توحيد الكود المنشور مع GitHub.
3. ضبط البريد SMTP/Resend.
4. إنشاء حساب اختبار جديد أو التأكد من وجود حساب أحمد على قاعدة الإنتاج.
5. إعادة اختبار الفلو:
   - إنشاء حساب
   - وصول رسالة التفعيل
   - تفعيل الحساب
   - اختيار الباقة
   - الدخول للوحة التحكم

## أوامر فحص مفيدة على السيرفر

```bash
cd /home/botzkprp/botzo.net
php artisan route:list | grep -i register
php artisan route:list | grep -i login
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

## توصية النشر التالية

الأفضل عدم تعديل `/home/botzkprp/botzo.net` يدويًا مرة أخرى. نحدد طريقة واحدة للنشر:

- إما نقل/مزامنة كود GitHub إلى `/home/botzkprp/botzo.net`.
- أو تغيير Document Root ليشير إلى clone من GitHub بعد تجهيز `.env` و`storage`.

حتى يتم ذلك، سيظل هناك احتمال أن الـ local/GitHub والـ deployed يظهروا بواجهات مختلفة.
